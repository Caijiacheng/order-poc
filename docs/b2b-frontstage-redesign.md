# B2B 前台产品重构

## 1. 目标

将当前“AI 建议单 + 智能凑单 Demo”的前台，重构为更接近标准 B2B 下单流程的经销商采购界面。

核心原则：

- 主线是采购，不是 AI 演示
- AI 是增强层，不是主导航主角
- 页面要符合经销商做单心智：补货 -> 选品 -> 校正订单 -> 确认提交

---

## 2. 我对 sub-agent 方案的 review

sub-agent 的大方向是正确的：

- 指出了当前页面更像 AI 控制台而不是 B2B 下单页
- 把 AI 从主入口降为增强层
- 把购物车改回订单优化页
- 把确认页改回交易确认页

但我认为它还需要 4 个关键修正。

### 2.1 经销商切换不能作为真实前台主动作

真实 B2B 场景里，经销商登录后通常只看到自己的账户，不会在前台主流程里反复切换 dealer。

因此：

- `切换经销商` 只能作为 demo mode 控件保留
- 正常前台应默认进入“当前经销商”的采购界面
- 如果为了演示必须保留切换，也应弱化到顶栏小入口或演示控制器，而不是页面主区第一个操作

### 2.2 “建议单”不应该是主导航，而应该是首页中的一个模块

标准 B2B 下单不应让用户先进入一个“建议单模式”，而应进入采购首页或采购工作台。

因此：

- 顶部导航不建议保留 `建议单`
- 建议改成：`采购首页` / `商品选购` / `采购清单` / `下单确认`
- “建议单”应退化成首页中的 `今日待补货` 模块或“系统建议补货”模块

### 2.3 标准 B2B 采购入口必须加入“常购”和“上次订单”

sub-agent 提到了“常购快捷补货”，但我认为还不够。更像 B2B 的入口至少应该包括：

- `今日待补货`
- `常购快捷补货`
- `上次订单再来一单`
- `本周活动/经营推荐`

这是经销商真实下单时最自然的四个入口，AI 建议应该混在这些入口中，而不是单独悬浮出来。

### 2.4 确认页必须像交易确认页，而不是优化摘要页

当前确认页即使重构，也容易只剩“本单优化说明”。这不够像 B2B 下单。

确认页至少应包含：

- 收货信息
- 配送日期 / 送达方式
- 结算方式 / 账期
- 发票信息
- 订单备注
- 订单明细与金额汇总
- 本单优化说明

即使这些字段是 mock，也必须出现，页面才像一笔标准交易。

## 3. 最终重构结论

我建议仍保留 4 个前台路由，但彻底重写它们的定位。

### 3.1 `/` 采购首页

页面定位：

- 经销商进入系统后的采购入口
- 不是“AI 建议单生成页”

页面模块：

- 顶部账户信息条
  - 经销商名称
  - 客户等级
  - 上次下单时间
  - 当前账期 / 结算方式
  - 当前采购清单摘要
- 模块一：`今日待补货`
  - AI 整理出的待补货建议
  - 每条显示：商品、建议数量、简短原因、加入采购清单
- 模块二：`常购快捷补货`
  - 高频 SKU 的快捷补货入口
- 模块三：`上次订单再来一单`
  - 复购型入口，符合 B2B 习惯
- 模块四：`本周活动专区`
  - 活动主推、新品铺货、组合搭售
- 侧边条：`本次采购清单`
  - SKU 数、件数、金额、距门槛差额、去选品、去购物车

AI 融入方式：

- 出现在 `今日待补货`
- 出现在活动商品的“为什么推荐”
- 不再出现“生成建议”这种主按钮

### 3.2 `/order` 商品选购工作台

页面定位：

- 经销商真正做单的主工作台

页面结构：

- 顶部筛选条
  - 全部商品
  - 常购
  - 待补货
  - 活动
  - 新品
- 左侧筛选区
  - 分类
  - 品牌
  - 规格
  - 搜索
- 中间商品区
  - 商品名
  - 规格
  - 单价
  - 起订量 / 箱规
  - 活动标签
  - 数量输入
  - 加入采购清单
- 右侧固定采购清单
  - 已选商品
  - 数量
  - 金额
  - 门槛进度
  - 去购物车

AI 融入方式：

- 作为商品标签：`建议补货`
- 作为侧栏提示：`你还缺 2 项常购`
- 作为区块提示：`系统优先建议补这几项`
- 不再保留“生成快速建议”作为主按钮

### 3.3 `/cart` 采购清单 / 订单优化

页面定位：

- 对采购清单做最后校正
- AI 在这里提供交易决策增强

页面结构：

- 左侧：购物车明细
  - 商品
  - 单价
  - 数量
  - 小计
  - 来源
  - 修改 / 删除
- 右侧：订单摘要
  - SKU 数
  - 件数
  - 总金额
  - 门槛状态
  - 预计优惠 / 预计免运费
- 右侧下方：订单优化建议
  - 金额优化：差多少可达门槛
  - 订单规范化：箱规修正、整箱建议
  - 搭配补充：已选商品的联动加购

AI 融入方式：

- 行内提示：建议整箱
- 右侧卡片：再补一点更划算
- 搭配推荐：已选 X，建议顺带补 Y
- 所有建议都可直接应用，不需要进入聊天

### 3.4 `/confirm` 下单确认

页面定位：

- 交易确认页
- 不是推荐追踪页

页面结构：

- 模块一：订单明细
- 模块二：收货与配送信息
- 模块三：结算与发票信息
- 模块四：订单备注
- 模块五：金额汇总
- 模块六：本单优化说明

AI 融入方式：

- 只保留本单优化说明
- 每条说明支持“查看原因”
- 不展示 `run id / trace / 推荐记录`

## 4. 推荐的前台导航改名

当前导航：

- 首页
- 建议单
- 购物车
- 提交确认

建议改成：

- 采购首页
- 商品选购
- 采购清单
- 下单确认

---

## 5. Demo mode 与真实经销商模式

建议把前台拆成两个认知层：

### 5.1 经销商模式

- 不显示 dealer 切换
- 不显示 run / trace / 推荐记录
- 更像真实 B2B 系统

### 5.2 演示模式

- 可显示 dealer 切换
- 可显示“查看推荐记录 / 打开 Langfuse”
- 仅用于客户演示，不属于标准经销商使用界面

这样可以同时兼顾：

- 真正的 B2B 采购体验
- 你做 POC 演示时的链路可视化

## 7. 最终判断

如果你的目标是：

- 让客户觉得“这真的像一个经销商下单系统”
- 而不是“一个套了下单壳子的 AI demo”

那么最关键的改动不是换视觉皮肤，而是：

1. 把 AI 从主入口移到采购流程内部  
2. 把“建议单”降成模块  
3. 把“采购清单”提升成核心中间态  
4. 把确认页补成标准交易页  
5. 把 dealer 切换和 trace 下沉到 demo mode

这 5 个动作一做，整个产品气质才会从“AI 展示台”变成“B2B 下单系统”。
